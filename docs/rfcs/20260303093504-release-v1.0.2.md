# RFC: Release v1.0.2 - Enhanced Hyphenation and Unit Testing

This release focuses on improving the robustness of the hyphenation handling, specifically for OCR-derived text, and establishing a formal testing suite.

## Staging

### ZID 20260303093504: Refine Hyphenation for `¬ `
Users observed that text extracted from certain PDFs or OCR engines uses the logical negation symbol (`¬`) followed by a space to indicate word hyphenation (e.g., `leuch¬ tet`). The utility was updated to recognize and remove this sequence.

### ZID 20260303094320: Implement Unit Tests
To ensure the reliability of the cleaning logic and prevent regressions, a comprehensive unit test suite was implemented using the Python `unittest` framework.

## Implementation Details

### Analytics
- **Hyphenation Patterns**: Analysis of user-provided OCR text revealed that the `¬` symbol is frequently used as a "soft hyphen" placeholder. Sometimes these are at line ends, but often they appear mid-line followed by a space due to how the PDF text layer is reconstructed.
- **Regex Limitations**: The previous regex `r'-\s*\n\s*'` was too narrow, only matching the standard hyphen at literal line breaks.

### Decisions
- **Unified Hyphen Regex**: Updated the primary hyphenation rule to `r'[-\u00ac]\s*\n\s*'`, covering both standard hyphens and the negation symbol at line breaks.
- **Mid-line Hyphenation**: Added a secondary rule `r'[-\u00ac]\s+'` to handle the `¬ ` case occurring on the same line, ensuring words like `leuch¬ tet` are correctly joined to `leuchtet`.
- **Test Structure**: Adopted a standard `tests/` directory structure with `test_remove_newline_util.py`. Test cases cover:
  - Multi-line hyphenation (Standard and `¬`).
  - Same-line `¬ ` hyphenation.
  - Newline normalization.
  - HTML tag stripping.
  - Whitespace and punctuation normalization.
- **Verification**: All tests pass, confirming the new logic works without breaking existing features like HTML cleaning or newline replacement.
