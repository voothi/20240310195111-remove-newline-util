# RFC: Release v1.1.0 - External Configuration Support

This release introduces external configuration support via `config.ini`, allowing users to customize hyphenation marks and cleaning behavior without modifying the source code.

## Staging

### ZID 20260303095043: Externalize Hardcoded Variables to `config.ini`
Users requested the ability to configure parameters like hyphenation marks externally. The utility now loads settings from a `config.ini` file, providing greater flexibility for different OCR engines and text sources.

## Implementation Details

### Analytics
- **Hardcoded Constraints**: Previously, hyphenation marks (`-` and `¬`) were hardcoded in the regex. While effective for most German text, other languages or OCR engines might use different symbols.
- **Configurability**: Users might want to toggle specific cleaning rules (like space collapsing) depending on their workflow.

### Decisions
- **`config.ini` Integration**: Added `configparser` to load settings from a local `config.ini`.
- **Configurable Hyphenation**: The `hyphenation_marks` setting allows users to define a custom string of characters to be treated as hyphens.
- **Feature Toggles**:
  - `join_same_line_hyphens`: Controls whether hyphens followed by space on the same line are joined.
  - `collapse_multiple_spaces`: Controls whether multiple spaces are reduced to one.
- **Graceful Fallbacks**: The utility remains functional without a `config.ini` by using built-in defaults (`-¬`).
- **Testing**: Expanded the test suite to 9 cases, specifically verifying that custom configuration settings are correctly applied and respected by the `clean_text` function.
- **Documentation**: Updated `README.md` with a new "Configuration" section.
